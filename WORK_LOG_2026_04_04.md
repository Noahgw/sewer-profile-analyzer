## Issue: Profile View Not Updating After Fix

### Problem

When a user applied a fix tool (e.g., Min Slope from Upstream), the Profile View tab's Plotly chart did not re-render with the updated invert values. The fix was applied correctly to the edit ledger, but the visual profile stayed the same.

### Prior Failed Attempts (from previous session)

1. **Read `ledger_len` in tab scope** — Tabs don't re-render inactive content.
2. **`solara.FigurePlotly(fig, key=...)`** — Crashed: `FigurePlotly` doesn't accept a `key` parameter.
3. **Made ProfilePanel a `@solara.component`** reading `edit_ledger.value` — Didn't update.
4. **Used `plotly.io.to_html()` via `solara.HTML()`** — Profile stopped showing entirely because Solara strips `<script>` tags from `unsafe_innerHTML`.
5. **`use_state` with version tracking** — Called `set_state` during render (anti-pattern in Solara).

### Fix Iteration 1: Inline Figure Computation

Simplified `ProfilePanel` to compute the figure inline during render and pass a fresh `go.Figure(fig.to_dict())` to `FigurePlotly`. This ensured a new figure object each time.

**Result:** Profile still didn't update.

### Fix Iteration 2: Explicit Dependencies

Investigated `FigurePlotly` source code and discovered it uses `solara.use_effect(update_data, dependencies or fig)` internally. Without explicit `dependencies`, it compared figure objects — and Plotly figures with identical data compared as equal via reacton's `utils.equals()`.

Added explicit dependencies:
```python
solara.FigurePlotly(fig, dependencies=[ledger_ver, sel_key, insp_key])
```

**Result:** Min Slope to Downstream worked, but Min Slope from Upstream still didn't show changes.

### Fix Iteration 3: Profile Clamping Bug (Root Cause)

Discovered the actual bug in `_build_profile()`:
```python
# This was overriding the fix!
if ds_mh_inv is not None:
    ds_inv = max(ds_inv, ds_mh_inv)
```

The profile clamped pipe inverts to never go below the manhole invert using `max()`. When `min_slope_from_upstream` **lowered** the downstream invert to create a positive slope, `max()` snapped it back up to the original manhole invert — visually undoing the fix.

`min_slope_to_downstream` worked because it **raises** the upstream invert, so `max()` had no effect.

**Fix:** Removed the clamping logic. Pipe inverts from the edit ledger are authoritative and should be displayed as-is.

**Result:** Both fix strategies now correctly update the profile.

## Key Technical Learnings

1. **Solara `FigurePlotly` internals:** Uses `use_effect` with the figure object as the dependency. Must pass explicit `dependencies=[...]` to force re-renders on external state changes.

2. **Reacton `utils.equals()`:** Compares lists element-wise, functions by `__code__` and closure cells, and falls back to `==`. Plotly figures with identical data compare as equal, preventing `use_effect` from firing.

3. **Profile clamping pitfall:** Clamping rendered values against original data defeats the purpose of the edit ledger. The ledger should be the single source of truth for display values.
